블로그 개설

2025년 10월 4일

블로그는 학창시절부터 꾸준히 만들어서 개인적인 글이나 정보 공유성 올리거나 했던 경험이 많았습니다.

이번엔 연휴도 껴있고 뭔가 만들어보고싶은 생각에 평상시 생각해온 아이디어를 프로젝트화 하는것보단 간단하게 해볼 수 있는게 무엇이 있을까 하고 고민하다 단순하게 블로그를 만들어보기로 했습니다.

자랑할 만한 건 아니지만 토이프로젝트들을 개인적인 기념비처럼 어딘가 담아두고 싶었는데, 이참에 그런 공간도 한번 만들어보았습니다.

블로그는 뭐로?

일단 요즘 cursor나 cluade-code 를 활용해서 개인 프로젝트 개발시에는 좀 활용하고있어서 지금 이 블로그도 도움을 많이 받았습니다. 사실 요즘 같이 바이브코딩이 가능해 진 덕분에 훌륭한 손발을 얻게 된것같네요.

블로그를 만들기위한 도구로 헥소나 지킬 등등 여럿 많지만 NextJS에서 안써본것도 좀 써보고싶단 생각도 들어서 막연하게 일단 NextJS로 만들어 보았습니다.

만들다보면 시간의 제약에 항상 아쉬운점이 보입니다. 다른 개발자 블로그들을 보니 재미있는게 많아보여서 차차 개선해보는것도 괜찮을 것 같습니다.

블로그를 만들때 다른것보다 중요한 기능들은 마크다운을 쉽게 파싱해서 활용하게 만드는, 부분이라던가 관련 설정인것같아 조금 내용을 추가해보면 어떨까 싶습니다.

markdown 파싱

마크다운으로 글을 작성하고 이를 HTML로 변환해서 보여주는게 블로그의 핵심 기능이라고 생각합니다. 이를 위해 두 가지 패키지를 활용했습니다.

gray-matter는 마크다운 파일의 메타데이터를 파싱하는 역할을 합니다. 블로그 글 상단에 있는 ---로 감싸진 부분(frontmatter라고 부르는 것 같습니다.)에서 제목, 날짜, 태그 같은 정보들을 추출해줍니다.

Marked는 실제 마크다운 본문을 HTML로 변환해주는 라이브러리입니다. 마크다운 문법으로 작성한 텍스트를 브라우저가 이해할 수 있는 HTML로 바꿔줍니다.

이 두 패키지를 조합하면 마크다운 파일을 읽어서 메타데이터는 gray-matter로 추출하고, 본문은 Marked로 HTML로 변환하는 플로우가 만들어집니다.

깃헙 페이지에 github.io 도메인 활용하기

GitHub Pages를 사용하면 기본적으로 username.github.io 형태의 도메인을 제공받습니다. 레포지토리 이름을 username.github.io로 만들면 루트 도메인으로 접근할 수 있게 됩니다.

설정 자체는 어렵지 않습니다. 레포지토리 설정에서 GitHub Pages를 활성화하고, 빌드된 정적 파일들을 올리면 자동으로 배포가 됩니다. NextJS로 빌드한 정적 파일들을 out 디렉토리에 생성한 뒤, 이를 GitHub Pages가 인식할 수 있게 푸시하면 끝입니다.

NextJS를 정적 페이지로

NextJS는 기본적으로 서버 사이드 렌더링을 지원하는 프레임워크이지만, Static Site Generation(SSG) 기능도 제공합니다. 정적 페이지는 빌드 타임에 모든 HTML을 미리 생성해두는 방식입니다.

NextJS를 정적 페이지로 만들기 위해서는 next.config.ts에 output: 'export' 설정을 추가해야 합니다. 이렇게 하면 빌드 명령어 수행 시 서버 코드 대신 순수한 HTML/CSS/JS 파일들이 out 디렉토리에 생성됩니다.

GitHub Pages에 배포하기 위해서는 빌드된 out 디렉토리의 내용을 레포지토리에 푸시하면 됩니다.

이때 블로그 개발용 레포는 숨겨두고 싶었습니다. out 디렉토리만 git submodule로 username.github.io 레포에 연결해서 블로그 개발용 레포(private)와 배포 레포(public)으로 분리하는 방법을 사용해봣는데, 나쁘지않은것같습니다.

아무튼 다시 돌아가서 블로그 만든 이유

여태까지 꽤 많은 개인 프로젝트가 빛을 보지못하고 접혀진 경우가 많았습니다.

실제론 업무가 바빠지는 부분 때문이 80%, 다른 관심사가 생겨서가 10% + 그외..

어쨋던 그것도 꽤 큰 토이프로젝트들이 있었으니 이제는 좀 담아주고싶다

그리고 AI 관련해서 공부 좀 더 해보고싶은 마음에 마음을 다잡고자 만든면도 있습니다

요즘 AI 시대 덕분에 귀찮음을 조금이라도 덜 수 있는 수단이 생겼으니. 또 앞으로 토이프로젝트를 진행하는데 필요한 체력을 AI 토큰으로 등가교환 할 수 있을 것 같습니다.
