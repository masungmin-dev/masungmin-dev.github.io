1f:Tbd9,<h1>소스코드는 숨기고, 결과물만 공개하고 싶다..!</h1>
<p>블로그를 만들다보면 GitHub Pages로 배포하는 경우가 많습니다. 그런데 문제가 하나 있습니다.</p>
<p><strong>소스코드가 모두 공개된다는 점</strong></p>
<p>물론 오픈소스로 공개하는 것도 좋지만, 블로그 소스코드에는 개인적인 설정이나 실험 중인 코드, 또는 아직 완성되지 않은 초안 글들이 포함되어있을 수 있습니다.</p>
<p>배포된 결과물만 공개하고, 소스코드는 private으로 관리하고 싶었습니다.</p>
<h2>Git Submodule</h2>
<p>Git Submodule을 활용하면 이 문제를 해결할 수 있어보였습니다. 예전엔 자주 썻는데, 모노레포로 워크스페이스를 관리하다보니 최근에는 잘 활용한적이 없네요.</p>
<p><strong>아무튼 이런식으로..</strong></p>
<ol>
<li>블로그 소스코드를 담은 <strong>private 저장소</strong> (예: <code>my-blog-private</code>)</li>
<li>빌드된 정적 파일들을 담을 <strong>public 저장소</strong> (예: <code>username.github.io</code>)</li>
<li>private 저장소 내부에 public 저장소를 <strong>submodule</strong>로 연결</li>
</ol>
<p>이렇게 하면 소스코드는 private으로 관리하면서, 빌드 결과물만 public 저장소에 푸시할 수 있습니다.</p>
<h2>실제 구조 살펴보기</h2>
<p>실제 디렉토리 구조에서 중요한 것만 봅시다..</p>
<h3>디렉토리 구조</h3>
<pre><code>blog-source/          (private 저장소)
├── ...
├── out/              (public 저장소 - submodule)
│   └── .git/
└── package.json
</code></pre><h3>Submodule 설정</h3>
<p><code>.gitmodules</code> 파일에 submodule을 등록해둡니다</p>
<pre><code class="hljs language-gitconfig">[submodule &quot;out&quot;]
	path = out
	url = git@github.com:내공개레포.git
	branch = deploy
</code></pre><p><code>out</code> 디렉토리가 실제 public 저장소를 가리키고 있습니다. 이제 빌드 결과물을 이 디렉토리에 넣고 푸시하면 됩니다</p>
<h2>배포 과정</h2>
<p>기본적으로 다음과 같은 흐름입니다</p>
<ol>
<li><strong>빌드</strong> - Next.js를 정적 사이트로 빌드</li>
<li><strong>복사</strong> - 빌드 결과물을 <code>out</code> 디렉토리(submodule)로 복사</li>
<li><strong>푸시</strong> - <code>out</code> 디렉토리에서 public 저장소로 커밋 &amp; 푸시</li>
</ol>
<p>저는 이 과정도 쉘 스크립트로 자동화를 해두었습니다. 귀찮으면 안되잖아요?</p>
<p>Private 저장소의 변경사항과 Public 저장소(submodule)의 변경사항을 별도로 관리하면 됩니다</p>
<h2>마치며</h2>
<p>Git Submodule을 사용하면 소스코드는 private으로, 배포는 public으로 관리할 수 있습니다.</p>
<p>물론 GitHub Actions나 Vercel 같은 다른 방법들도 있지만, 저장소를 명확하게 분리하고 싶다면 Submodule도 괜찮은 선택지입니다.</p>
